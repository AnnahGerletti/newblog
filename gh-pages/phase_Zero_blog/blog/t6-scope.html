Scope and Variables

June 22, 2017

What's 'scope', and how does it work in JavaScript.

JavaScript Scope deals specifically with variables. Objects and peramiters are also variables.

Scope describes the amount of access you have to the variables.

Scope can be globally and locally defined.

Global scope is very simple â€“ variables that are set outside of functions are globally known.

Local scope - is created by a function.

For example:

function myName(name){
        var last = 'Smith';
        console.log(myName + last);
      };
      // John Smith

The variable last is only defined within the myName function meaning this variable becomes locally scoped. If var last was called outside of the function there would be an error message saying var last is not defined.

Here is an example of variables that are defined within nested functions:

function showMe(name){
        var message = 'Help!';
        if(name =='John'){
          lookLeft('Steve');
        }
        function lookLeft(player){
          console.log(name + 'is running from' + player);
          console.log(player + 'says' + message)
        }
      }
      // John is running from Steve
      // Steve says Help!

There are three variables defined above, name, message, player. The variable player nested inside the lookLeft function is locally scoped and can only be called from within that specific function. Variables name and message are locally scoped but can be called in both the showMe and lookLeft functions.
